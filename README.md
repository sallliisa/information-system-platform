# Base Frontend

Admin frontend built with Vue 3 + Vite + TypeScript.  
This project uses dynamic module routing, Pinia state management, Tailwind CSS, and integrations like Google Maps, charts, and rich text editing.

## Tech Stack

- Vue 3 (`script setup`, async components)
- Vite 4
- TypeScript
- Vue Router (hash history)
- Pinia
- Tailwind CSS + PostCSS
- Vitest + Vue Test Utils
- ESLint + Prettier

## Requirements

- Node.js 18+ (recommended)
- npm (default) or bun

## Getting Started

1. Install dependencies:

```sh
npm install
```

2. Copy environment file:

```sh
cp env.sample .env
```

3. Set required env value:

```env
GOOGLE_MAP_API_KEY=your_api_key
```

4. Run development server:

```sh
npm run dev
```

5. Open the app at:

```txt
http://localhost:5173
```

## Available Scripts

- `npm run dev` - start Vite dev server
- `npm run build` - build for production
- `npm run preview` - preview production build on port `3100`
- `npm run test:unit` - run unit tests with Vitest (`jsdom`)
- `npm run type-check` - run type checking via `vue-tsc`
- `npm run lint` - run ESLint with autofix

## Environment Variables

- `GOOGLE_MAP_API_KEY`: used by map components (see `MapView.vue` and location inputs)

Vite is configured with:

```ts
envPrefix: ['VITE_', 'GOOGLE_MAP_API_KEY']
```

So this project accepts both standard `VITE_*` vars and `GOOGLE_MAP_API_KEY`.

## Project Structure

```txt
src/
  app/                # app-level configs, langpacks, and action schemas
  components/         # base UI, composites, inputs, navigation
  layouts/            # authenticated, unauthenticated, public, blank
  router/             # route creation + auth guard
  stores/             # Pinia stores
  utils/              # API/service, storage, auth, helpers
  views/              # route view pages
```

## Runtime Configuration Notes

- Active app mode is set in `src/mode.ts` (currently `development`).
- API base URLs and app keys are defined in `src/config.ts`.
- Company code is set in `src/company.ts` (currently `HK`).

If you need different API targets per environment, update these files or migrate the values to `.env` variables.
